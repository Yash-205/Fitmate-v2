# Code Commenting Guidelines

When writing code, adhere to the following rules for comments:

1. **Scope:** Each comment should explain the logic for the next 5-6 lines of code.
2. **Brevity:** Keep regular comments restricted to a single line. They must be brief and concise.
3. **Functions and Components:** Exceptions are made for function definitions or React component declarations, where multi-line comments are permitted to explain the overarching purpose or props.
4. **Coverage:** While each logical block of code can be explained, it is not strictly necessary to comment every single block—only where the intent needs clarification.
